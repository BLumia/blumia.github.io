TimeCapsule - 2017

于是现在该写17年的时间囊辣

鉴于开去年“时间囊”的时候都忘记上次时间囊写的时候我在想些什么了，于是这次干脆分条带原因写得了。于是下面是内容。

前端特技化工具链

首先想到的是打算系统的学一下前端科技，前端工程化的那堆东西，node.js啊webpack啊babel啊ES6啊什么的。原因两个，一个是jxjxj学这个的确吸引到我了（看着就好厉害），另一个就是翻github repo的时候会看到大家都在用，也就我还在写那种一点也不优雅的意大利面htmlcssjs代码。

当然，如果一定要说还有别的原因的话，那就是因为班里有一些人开始啃html+css+js了，学新的是为了能再度把距离甩远。至于以后会不会从事前端方面的工作，不敢断言，但是我意向上不想做这个..

顺便，这个似乎不算纯零基础，之前的野路子还是用过点Node.js的

C++从入门到放弃

这是要把复习c和c艹提上日程了。因为距离毕业愈加接近了，我觉得我大三才意识到刷绩点的重要性我觉得我之前扔掉那么多，考研基本无望了，于是必须得重新拾一些基本功以后面试才有底气，于是得重新啃一下C艹。

至于为什么是C艹，首先我有一些脑洞大概会选C艹去实现，比如之前写了一半的PineappleSynth的效率问题还没来得及填坑，以及没准我还真会去写个Midi Sequencer？别的的话，我还有个坑是实现一个跨平台的简单图形api，这个没点底子我估计我连个wrapper都写不好。哦对，我现在连个非托管式的dll都没写过…

于是肯定会做的就是重新学C艹，造点轮子练手，大概还会顺带学一下C语言的一些写法，顺带估计能补点操作系统什么的坑。

一门生产力语言

还记得我高中的时候比较喜欢玩Lazarus，一个RAD IDE我觉得我能玩那么久，而现在我却没什么东西可以像之前那样得心应手（之前得心应手吗？）的玩了，于是为了有个能像之前那样用着顺手的东西玩，我应该会去学一套能做RAD的东西，比如C#/WinForm，或者别的合适的选择。

至于不直接选Qt的原因是我觉得那个似乎不太方便，不过我应该也会顺带再去熟悉一下QtQuick那一套东西。哦对了，Electron大概也会试一下，鉴于拿框架瞎搞前端现在还玩的挺高兴，用着舒服的话大概以后也会常用。

当然，五花八门的选择，关于考虑必学的还得是匸井和Lava中的一个，Electron什么的即便真的方便上手天天用了也算在学着玩的列表里。

硬件知识

和某硬件专家基友(?)聊天总能看到各种各样的硬件型号冒出来而我却不知道是什么只能赶紧开个鼓哥搜一搜，而除此之外我水个群都能看到讨论各种显卡啊主板啊什么插槽啊什么接口啊第几代啊马甲卡啊什么的[1]，而就即便我不水群,和同学讨论个手机也是什么什么型号cpu怎么怎么样，于是感觉我这硬件盲盲的太厉害了，是得补补各种硬件知识了。

讲道理我也不知道到哪个程度算可以，不过至少别人冒名词的时候我得知道是在说什么…或者定一个炒鸡低的目标：让我自己写配置单子我得会写…我现在甚至都不知道电源功率和我选什么硬件到底有什么关系…

电脑音乐制作

这个的原因…本身打算写乐理的，然而这部分内容不止是乐理，于是写了“电脑音乐制作”。想起我去年网易云音乐人审核传的那个曲子播放量还不如我拼loop出来的曲子我就觉得很蛋疼…于是我大概会好好学一下电脑音乐制作。

不过毕竟这不是什么正经事儿，于是大概还是不会有大片的时间学这个，所以我大概会做一些OHC（OneHourCompo，一小时挑战），线上的或者自己找话题的，然后曲风大概会先限制在cliptune上（对电子乐炫技要求不是很严格，大概不会有成屏幕的自动控制，也不用堆音色，以及某种形式强制应用乐理知识以免偷懒放水）

如果能在bkt里搞OHC比赛大概会很好玩？

绘画？

我又双叒叕在时间囊里写“绘画”了，鬼知道这回是不是又会预测失败。

于是更进一步来说，今年应该会无论如何做出尝试，然后看看到底能画成什么样。[2]

这段真不知道该写点啥，至今没个roadmap的…哦对，预测今年会有自己的绘画设备[3]

我觉得我还会

  • 学习一些软测知识
  • 接触一门服务器开发用的语言
  • 入门3D建模相关的知识
  • 完工BLumiaOJ并弃坑
  • 发布至少三个单曲
  • 抽时间整理我硬盘上的乱七八糟文件
  • 学会说话
  • 有自己的设定(?)

上面这四条算是预言吧，软测知识是我觉得我搞github项目的话用CI得顺带学的，服务器开发语言我觉得大概会便于我写一些奇怪的轮子，但是我也不敢确定我到底会不会去学，3D建模知识入门我也不清楚会不会排得上日程但是我觉得我也许还是会抽时间去学的。BLOJ这个，我觉得BLOJ在能用的程度后会停坑[4]。单曲的话就是说有三个完整曲目而已。后面几条不予解释。

我觉得大概不会

  • 学习SSH框架
  • 开坑新滚球游戏计划
  • 女装

SSH框架是真恶心到我了，我觉得我不会去学这个。这个预测的原因是学校和某培训机构合作有教这个的。至于新滚球，我觉得不会拍脑袋再说做了。不过我也许会把我的脑洞试着实现成可玩的游戏，可能性非常小吧，毕竟就这么点技能。第三条我确定我不是在立flag！

额外的话

我大概会好好整理一下我的域名和网站，现在我连某域名的泛解析和www解析都刻意没做，以及乱乱的服务器和网站页面…另外我今年内大概会再见Chris一次？

2016年末小结

去年的时间囊开囊是15年12月末，而现在已经是17年1月9号了。不过也还好，今天期末考试结束，这学期也算完结了，趁此时机做一下16年小结。

于是，最主要的，先看一下去年的时间囊的状况吧。

编程语言：

  • python
  • c#
  • node.js
  • bash
  • Haskell

!!! Result

现在想来这些都没系统的学，于是这四个其实算起来都没学。不过似乎我长久以来都没有系统的学过什么东西了，于是如果要真去学一个东西，还是得系统的学才能学到真实用的吧，野路子注定写的一团糟（？）

python只是看过网上的教程，没跟着写什么代码，不过玩过django去写我的一个脑洞网站，当然也只是玩玩，现在这个repo被我晾着了..

说要复习cpp并没有复习，于是这部分的时间囊几乎一个都没命中（于是我今年在干什么..）

学习内容：

  • Modern OpenGL
  • GLSL / Cg
  • Linux
  • DSA
  • Music Theory
  • Drawing

!!! Result

于是这块儿比较神奇，分开说吧。

OpenGL这部分去年年初就开始学了一段时间，那段时间我还攒了好久的 GitHub commit 连击（虽然后面感觉激励写代码的味道变了后就断掉了），感觉是GL的确挺好玩不过也挺难的，比如我可以轻易的写出不知道问题在哪的玄学shader…当然GL也没学太多就停了，一方面是因为玄学一般的shader，另一方面好像是因为后来开学了事情有点多就放一边了。

当时分开写了OpenGL和GLSL，感觉学的话似乎总得一起学？至于学以致用的事情，除了自己写的程序能用到自己写的shader，我还真没研究过游戏引擎里的那一套机制，于是ShaderLab啊Cg啊什么的我还是一概不了解的状态。

Linux我几乎是作为主力环境长期使用了6个月多，于是这个过程自然学了一些东西，不过不多，毕竟我一直是桌面环境用户，不去主动挖的话和用windows一样学不到什么额外的东西的。不过，terminal是真比windows的cmd舒服。

至于后来，Linux我也一直有在用，不过回到windows的情况多一些，最初原因是要魔改某个神奇东西得用Visual Studio写c#，后来还颓废了一段时间（看我GitHub contributions表就知道哪段时间我颓废了）..

DSA… 我的确没抽出时间专门去回顾这些:(

然后是乐理。乐理这个我似乎野路子学了一些，或者说，乐理书没好好看，看好和弦和别的零散视频和资料学了点。因为乐理底子不算零蛋，所以现在野路子学也不算非常奇怪。毕竟整个一年我也没抽出大片的时间专门给学乐理这个东西。

声音合成方面倒是学了一些，一方面拜Syntorial所赐，另一方面就是我自己用WDL-OL这个wrapper写了个just works的vsti虚拟乐器，其间学了一些。另外的话就是瞎玩各种DAW学的了。

DAW几乎把常见且不肥的用了个遍，sonar我有正版于是在用[1]，感觉pianoroll挺不舒服的，而且sonar steam少一些功能（pro才有的pro channel啊，一些插件之类），不过TTS-1和Z3ta+2挺好的。fl也比较常用，用着还算比较顺手，不过目前是“共产版”[2]，Ableton live我买launchkey 25key mk2的时候送了lite，体验感觉还行，但是总觉得工作流怪怪的不太顺手。studio one的那个免费版对于电脑音乐制作几乎是不能用的程度（连第三方vst都不让挂我玩什么），cubase鉴于懒没怎么玩… 于是结论是，sonar和cubase那样比较经典的工作流我还是很习惯的，而fl studio的钢琴卷帘太顺手了，虽然工作流奇怪一些，但是最终我还是用fl比较多，而sonar目前是我专门用来编辑/转换mid到mp3用的- -

最后是画画，也没进度，因为没看到比较完备的适合我的目标的路线图。毕竟目的非常“清真”，就是画画“小动物”[3]，然后这个目的和我看到的….上来画素描？蛤？不过无论如何，虽然没有任何进度，不过我觉得还是要付出行动尝试一下比较好，于是今年的计划肯定还会有画画，而且有一些具体的目标。哦对了，我没有购买61的二手数位板[4]。

软件：

  • Unity 3D
  • FL Studio
  • blender

!!! Result

Unity 3D这个因为之前提到的老周一事必须得写一些代码，几乎不算是主动学的，不过多少还是学了一些，清楚了这个东西的工作流和脚本组件机制。然后我觉得还是系统的学一下c#比较好。另外魔改osu!代码的时候也写了c#，大概也算学了一些c#？

fl这个上面提到了，结果上面也提到了，入了midi键盘，而且的确学了些DAW的使用和音频合成方面的东西，这块儿时间囊算是准的吧。

blender这个，看过些教程，忘得比看正则忘的还快。3D建模方面我目前还算是0基础的吧，于是也不算学了。

小工程：

  • RumidiaPlayer
  • SoftwareExplorer
  • BLumiaOJ

第一个是个midi播放器，开始闷声写了个midi读取就没再写了，Chris写了个QMidiPlayer，如果用的话我建议用这个。至于我的那部分代码，如果我打算搞midi编辑器的话估计还会用上（？

第二个就是现在的BearKidsTeam/Awesome了。这个算是清单和应用发现家的整合，不过鉴于django/python初心者，目前被晾着了，但这个也算我做了对吧？

BLumiaOJ这个的确被我晾了一年才拾起来，而拾起来的原因是因为学校打算商业化学校的c语言教学平台，其中包括OJ这部分。

虽说这是我拾起来的原因，但我拾起来并不是为了商业化和分一杯羹，我只是觉得填完坑给学校换上算是完事儿了吧，鉴于最初代码写的质量堪忧[5]，现在维护感觉很蛋疼，不过好在差不多能用了，于是“上线使用”的工作量应该不算大。另外，就在前几天，我整理了一下repo，分开了我自己完整重写的web部分，和依然被依赖的hustoj判题核心。大概也就没有license方面的问题了吧。

额外的话，去年时间囊最后一段提到了HR2的开发，这个反而和预期一样，现在又双叒叕没动静了。不过我个人觉得这很正常，一方面，三分钟热度的拍脑袋是做不出东西的，另一方面，BKT本身就是一个兴趣相投的人聚在一起谈天说地的地方，现在你说组个团搞个东西，大概还不太现实吧，不过若是真的可以能一起做些喜欢的东西，应该是不错的事情吧[6]。

最后关于16年所做的事情，我觉得我似乎的确没做什么特别的事情，计划外的工程有个Pineapple Synth，还有个超小号的Private Cloud Music。16年初还参与了LearnOpenGL网站的翻译，别的似乎的确没做什么。

一点人生经验的话，要真想学东西，还是系统的学习取得的价值更大一些，野路子的确也能学到东西，但是大概经常让我学出个半吊子，说会但不会说不会懂一点的程度吧。于是在我下次学东西的时候，大概会去找个完整的教程去啃，而不是去找网上的“跟我学xxx”/“实战xxx”博客文章去看。

差不多就是这些了，接下来休息一下，然后下一篇就是17年的时间囊了。以上。

December 29, 2015

2015年末小结

转眼就到了年末，是时候小结一下今年的得失了。也换一种方法来开所谓的时间囊。

于是当我打开15年时间囊的页面的时候，我发现我说“九木集性质的文章不会再发上来了”，然而我还是发上来一些…不过还好…该在硬盘呆着的还在硬盘呆着呢..这种杂集文章发出来自己会感觉好一些，以后翻过来还能看看自己之前干过的傻事儿。至于别人看到不看到，其实本来就没打算让很多人看到，能到这里的应该都是关系非常好的朋友了吧。所以看到了应该也没什么吧。。不过，或许真的有一天我会把我记水事儿的文章统统收回硬盘然后只留个什么所谓的技术博客？

接着看时间囊，那么下面也分块写。

1. Adobe常见软件的使用


老实说，我Ps用的更熟练了，虽然依然不会玩什么高级花样。总体对我来说，这些软件并不是我的硬性需求，所以我也没怎么专门去学/瞎搞。不过说起来，我现在主力系统换成linux后，编辑图片的时候还是会“想念”Ps。。。哦对了，Ae和Pr的workflow大致了解了，也大体会做一些简单的操作了，大致和我会Ps的程度一致…
ps. Vegas比Pr好使啊..

## 2. 其他常见软件使用


前俩实际都是指Git..由于目前我还是经常会选择给我的乱搞的小东西加上vcs，所以git的很多操作还是会的。不过命令倒是不会多少，也就能commit和push的程度（
无论是主力系统换成linux，亦或是别人安利，Vi/Vim我多少会了一些基本操作，不过这些操作都很简单，只是navigation的那部分。高级的…至少目前没怎么玩过。
水果（fl）的workflow大体也熟悉了，不再像之前一样只会在能看到的地方乱拖乐器瞎搞了。Ableton Live这个安利的新软件我还不怎么会玩，行走蛋糕依旧只会拿来当个midi编辑器用，哦对了。。圣诞特惠（现在依然在持续其实..）行走蛋糕在sbeam只要一百多，当时看到的时候真的想买。。。
Unity3D这个坑比较有意思，本来打算明年（大三/大二下半学期）再开坑，结果某一天上网的时候突然发现unity3d出linux版了（这时主力系统已经转linux了），异常兴奋，于是下载了，并开始慢慢了解unity5的各种姿势。不过。。linux上写c#真不爽啊。。

## 3. 新的编程语言


Java我有课所以不管这坑开不开都是要学的。不过尽管有课但是我依然没在平时写过java代码倒是真的。。C#则写了一部分。因为上面提到的Unity3d的坑开了的缘故才写的。但，java和c#根本没有网上说的百分之多少的相似度好伐？M$有好多黑科技语法糖什么的呢…至于Lua，这坑压根没开，Py反而开了但也没学多少（只是看了语法规则。。），相反倒是正经学了学node.js…

## 4. 旧知识的复习


自己很久都没搞什么自己的小项目了，所以我觉得我也没怎么复习cpp和oop。虽然好歹有个思想，学java的时候也复习了一部分。数据结构的话，有课，所以必须学，不过还是那么些东西。至于算法，或许忘得更多吧，不过这个毕竟会用上，会去复习并学些新的算法的。

## 5. 工程和项目化


不得不说HappyRoll这个让我可以说“爱过”的MarbleBlast Mod我停工了，有一次有人提醒我说论坛那边问是不是dead了，于是我过去发了个目前suspend了。的确，我觉得学点东西之后才能搞出更好的mod，但是实际上我不太想继续这个mod了，或许应该换个引擎，搞个新的，免费的，awesome的滚球游戏出来。说到底，滚球游戏才是能让我答“爱过”的东西。额，我想表达点什么东西但是不知道该怎么写出来了。想到了的话再补吧，或者以专题写出来。
额..U3D滚球这个在上一自然段描述过了…
至于ToK这个，其实我没兴趣，不过依然值得一写。说实话这是我第一次见识到什么叫“就差一个程序员”系列，以及目前的所谓创业圈到底多奇怪。（个人声称是公司并）拿着一个只有游戏流程设计的构思然后说着“我们正在搞”就可以拿到奖，我已经猜到创业圈有多奇葩了。出去参加创业比赛可以和名人谈笑风生，的确，但是又有什么用呢？[1]
安卓开发没入坑，放到明年了。ios算了，没设备。至于wxWidget。。我反而去看了gtk…


## 6. 网站开发部分


hustoj前端重写，实际上也就是BLOJ了，因为是完全重写所以并不是hustoj的什么fork。不过目前我没动力去写这个了。。因为我发现似乎根本没人会用。。至于贴吧仿制app我只是拿php搞过一个相当粗糙的版本。目前不打算搞了，我觉得即便要搞也得上mvc或者用一些科学的办法吧。另外就是。。我觉得php太蛋疼了。。
个人软件发布app我目前没有这个需求了。所以不再打算搞，不过我有个搞社交化软件/资源共享平台的小脑洞，这个新坑留着。至于个性化个人博客，我觉得我做的过头了，这一年都从wp到typecho再到hexo了。。。。

## 7. 其他内容


日语这个我本来报了下一学期（大二下）的选修，然而没选上。至于乐理，本来以为上音乐鉴赏会讲一些，谁知道只是讲了讲五线谱就完了。不过下学期我报了电子钢琴。。至于绘画。我觉得这坑开不了，因为连个基本的路线都没有。无从下手吧。

至于本年总结性质的东西这次就不写了。

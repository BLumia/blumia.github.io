2016年末小结

去年的时间囊开囊是15年12月末，而现在已经是17年1月9号了。不过也还好，今天期末考试结束，这学期也算完结了，趁此时机做一下16年小结。

于是，最主要的，先看一下去年的时间囊的状况吧。

编程语言：

  • python
  • c#
  • node.js
  • bash
  • Haskell

!!! Result

现在想来这些都没系统的学，于是这四个其实算起来都没学。不过似乎我长久以来都没有系统的学过什么东西了，于是如果要真去学一个东西，还是得系统的学才能学到真实用的吧，野路子注定写的一团糟（？）

python只是看过网上的教程，没跟着写什么代码，不过玩过django去写我的一个脑洞网站，当然也只是玩玩，现在这个repo被我晾着了..

说要复习cpp并没有复习，于是这部分的时间囊几乎一个都没命中（于是我今年在干什么..）

学习内容：

  • Modern OpenGL
  • GLSL / Cg
  • Linux
  • DSA
  • Music Theory
  • Drawing

!!! Result

于是这块儿比较神奇，分开说吧。

OpenGL这部分去年年初就开始学了一段时间，那段时间我还攒了好久的 GitHub commit 连击（虽然后面感觉激励写代码的味道变了后就断掉了），感觉是GL的确挺好玩不过也挺难的，比如我可以轻易的写出不知道问题在哪的玄学shader…当然GL也没学太多就停了，一方面是因为玄学一般的shader，另一方面好像是因为后来开学了事情有点多就放一边了。

当时分开写了OpenGL和GLSL，感觉学的话似乎总得一起学？至于学以致用的事情，除了自己写的程序能用到自己写的shader，我还真没研究过游戏引擎里的那一套机制，于是ShaderLab啊Cg啊什么的我还是一概不了解的状态。

Linux我几乎是作为主力环境长期使用了6个月多，于是这个过程自然学了一些东西，不过不多，毕竟我一直是桌面环境用户，不去主动挖的话和用windows一样学不到什么额外的东西的。不过，terminal是真比windows的cmd舒服。

至于后来，Linux我也一直有在用，不过回到windows的情况多一些，最初原因是要魔改某个神奇东西得用Visual Studio写c#，后来还颓废了一段时间（看我GitHub contributions表就知道哪段时间我颓废了）..

DSA… 我的确没抽出时间专门去回顾这些:(

然后是乐理。乐理这个我似乎野路子学了一些，或者说，乐理书没好好看，看好和弦和别的零散视频和资料学了点。因为乐理底子不算零蛋，所以现在野路子学也不算非常奇怪。毕竟整个一年我也没抽出大片的时间专门给学乐理这个东西。

声音合成方面倒是学了一些，一方面拜Syntorial所赐，另一方面就是我自己用WDL-OL这个wrapper写了个just works的vsti虚拟乐器，其间学了一些。另外的话就是瞎玩各种DAW学的了。

DAW几乎把常见且不肥的用了个遍，sonar我有正版于是在用[1]，感觉pianoroll挺不舒服的，而且sonar steam少一些功能（pro才有的pro channel啊，一些插件之类），不过TTS-1和Z3ta+2挺好的。fl也比较常用，用着还算比较顺手，不过目前是“共产版”[2]，Ableton live我买launchkey 25key mk2的时候送了lite，体验感觉还行，但是总觉得工作流怪怪的不太顺手。studio one的那个免费版对于电脑音乐制作几乎是不能用的程度（连第三方vst都不让挂我玩什么），cubase鉴于懒没怎么玩… 于是结论是，sonar和cubase那样比较经典的工作流我还是很习惯的，而fl studio的钢琴卷帘太顺手了，虽然工作流奇怪一些，但是最终我还是用fl比较多，而sonar目前是我专门用来编辑/转换mid到mp3用的- -

最后是画画，也没进度，因为没看到比较完备的适合我的目标的路线图。毕竟目的非常“清真”，就是画画“小动物”[3]，然后这个目的和我看到的….上来画素描？蛤？不过无论如何，虽然没有任何进度，不过我觉得还是要付出行动尝试一下比较好，于是今年的计划肯定还会有画画，而且有一些具体的目标。哦对了，我没有购买61的二手数位板[4]。

软件：

  • Unity 3D
  • FL Studio
  • blender

!!! Result

Unity 3D这个因为之前提到的老周一事必须得写一些代码，几乎不算是主动学的，不过多少还是学了一些，清楚了这个东西的工作流和脚本组件机制。然后我觉得还是系统的学一下c#比较好。另外魔改osu!代码的时候也写了c#，大概也算学了一些c#？

fl这个上面提到了，结果上面也提到了，入了midi键盘，而且的确学了些DAW的使用和音频合成方面的东西，这块儿时间囊算是准的吧。

blender这个，看过些教程，忘得比看正则忘的还快。3D建模方面我目前还算是0基础的吧，于是也不算学了。

小工程：

  • RumidiaPlayer
  • SoftwareExplorer
  • BLumiaOJ

第一个是个midi播放器，开始闷声写了个midi读取就没再写了，Chris写了个QMidiPlayer，如果用的话我建议用这个。至于我的那部分代码，如果我打算搞midi编辑器的话估计还会用上（？

第二个就是现在的BearKidsTeam/Awesome了。这个算是清单和应用发现家的整合，不过鉴于django/python初心者，目前被晾着了，但这个也算我做了对吧？

BLumiaOJ这个的确被我晾了一年才拾起来，而拾起来的原因是因为学校打算商业化学校的c语言教学平台，其中包括OJ这部分。

虽说这是我拾起来的原因，但我拾起来并不是为了商业化和分一杯羹，我只是觉得填完坑给学校换上算是完事儿了吧，鉴于最初代码写的质量堪忧[5]，现在维护感觉很蛋疼，不过好在差不多能用了，于是“上线使用”的工作量应该不算大。另外，就在前几天，我整理了一下repo，分开了我自己完整重写的web部分，和依然被依赖的hustoj判题核心。大概也就没有license方面的问题了吧。

额外的话，去年时间囊最后一段提到了HR2的开发，这个反而和预期一样，现在又双叒叕没动静了。不过我个人觉得这很正常，一方面，三分钟热度的拍脑袋是做不出东西的，另一方面，BKT本身就是一个兴趣相投的人聚在一起谈天说地的地方，现在你说组个团搞个东西，大概还不太现实吧，不过若是真的可以能一起做些喜欢的东西，应该是不错的事情吧[6]。

最后关于16年所做的事情，我觉得我似乎的确没做什么特别的事情，计划外的工程有个Pineapple Synth，还有个超小号的Private Cloud Music。16年初还参与了LearnOpenGL网站的翻译，别的似乎的确没做什么。

一点人生经验的话，要真想学东西，还是系统的学习取得的价值更大一些，野路子的确也能学到东西，但是大概经常让我学出个半吊子，说会但不会说不会懂一点的程度吧。于是在我下次学东西的时候，大概会去找个完整的教程去啃，而不是去找网上的“跟我学xxx”/“实战xxx”博客文章去看。

差不多就是这些了，接下来休息一下，然后下一篇就是17年的时间囊了。以上。

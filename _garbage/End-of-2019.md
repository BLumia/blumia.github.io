## 2019 时间囊部分

### 目标方面

不得不说其实今年非常惨，列的那么点目标啥都没完成。。。

#### 开始一个个人项目并完成它

> 当然这个目标计划的并不是那种下图工具之类的东西而是挖稍微大一些的坑，所以完成当然指的是达到一个差不多说得过去的 “1.0” 状态。至于可选的坑其实挺多的，不过这里就不再列出来了...

事实上 19 年内的确是有做了一些东西出来，比如这个 [PineapplePictures](https://github.com/BLumia/PineapplePictures) 辣鸡看图，以及一个 kdeconnect 接口的 Qt 绑定（对 DBus 接口的封装）。前者原因是我的确没有个顺手的看图工具，然后无论在 Windows 还是各 Linux 发行版下用的看图工具都多少有一些地方很蛋疼，难受到一定程度后忍不了了就写了那个东西。后者原因是我在 Windows 和 UOS/Deepin 下没法比较方便的看我的手机未读通知。当然这两个的完成度其实都达不到所谓 “1.0” 状态（后者目前的完成度甚至不好意思把代码扔到 GitHub），于是大概不能算是完成了这条计划。

不过说起来的话，最初我是打算试着开一个 [DTK 终端](https://github.com/BLumia/deepin-terminal-ng)的坑来玩，并且也的确做了一段时间[^1]，可惜后来公司决定加班赶 v20 版 DDE 之后我就再也没有时间继续做我的稍微大一些的 sideproject 了。有意思的是，后来公司也开始计划搞 v20 版本打算重制一个 DTK 终端出来[^11]，然后看上去似乎现在的终端开发组会拿着我那套代码继续开发，于是觉得我仿佛无形之中白白无偿加班了...

#### 三个曲目

> 但希望至少能够通过这个让我做点东西出来，多少也会有点长进。

这一条的目的是督促我做东西来练习，然后我年初想着等下半年再搞，结果下半年开始要求加班赶版本，就没时间搞了...不过还是有教训要吸取的，计划赶不上变化，以后能先搞的事情还是不要拖太久，指不定会发生什么...

#### 初步入门下面至少一样东西

> - 使用 Blender 的建模
> - 入门机器学习
> - 熟悉一个游戏引擎
> - 画画

在知道 blender 2.8x 版本会是大更新之后，最初就一直在留意相关的一些信息，也跟着官方放出来的测试版在玩，于使中间多少学了一些东西，但仅局限于基本操作。它正式发布 2.8x 版本好像就已经八月份了，然后上面提到了好多次的加班赶版本，于使然后就没有然后了...当然不得不说 2.8x 的变更的确让这东西的上手难度降低了一大截（至少对于我来说不用干点啥都要查文档了），然而我 19 年并没有真正意义上的入门这个东西..

ML 则是大致看了一点相关的资料（油管上的鼓哥官方的视频和一些别的地方的资料），然后那段时间觉得我似乎没有什么需求需要用 ML 或者相关的技术来做，于使就没有继续学下去了。

后面两条则是压根没开始，也就没得说了。

### 事情发展方面

#### 新的租房位置

> 这个感觉是一定会发生的事情，由于对当前租房的地方的电费收费方式的不满，我决定不会再续租当前住的地方了，于是接下来可能会选自如或者继续寻找个体租房。

我现在在自如内我的贼小的房间内写这篇年末小结...

原房子退租的时候大概是我 19 年最生气的一天，因为异常的电费，我冬天甚至不敢开空调取暖，但最后退租那天我却还需要付个几百块来补电费（最初租房是押一付三）。后来退租当天和房东吵了一架，甚至我报了警，然而警方那边说不管这个事，走的时候还对房东说了句“小心点别把事弄大，不然你也别想继续租房子”，可惜当时人傻没直接说这是违法出租要举报...

当然，后来在自如之后，水电费之类的就正常多了，尽管房租也贵了不少。

> 当然，既然写在这里那还是给个期望好一些，或许 chris 会来

Chris 后来的确决定来 D 司实习了，但现在反而有点后悔让 chris 来。和大多数我所知道的(想)来 D 司的人一样，大家都觉得 D 司是国内做开源贡献相关事情的为数不多的比较好的公司（所谓“情怀[^2]”），Chris 来也是打算看看能不能改善一下目前相关缺陷的地方（想做基础设施），结果一是并没有让 chris 做其想做的事情，二是实习工资比 L 司实习最低薪水还磕碜。

Chris 作为我一直以来的“偶像”和“试图追赶的目标”之一，本配得上去更好的公司实习，在更好的环境学习并具备更丰富的经验，却拿着少得可怜的薪水来做着并不配得上所期望的“情怀”的工作。尽管我很想和 Chris 说声抱歉，但直到 Chris 实习结束离开 D 司，我都没有说得出口，我也不知道为什么...

#### 购买 FL Producer Edition

> 上面“三首曲目”的附带相关。我可能会因此买掉 Buggy Studio 的 Producer Edition，原因是 Fruity 版这种不让处理音频轨的限制是真的非常蛋疼

因为这个打算所以我盯了一段时间的 FL 国内唯一的代理商（辣鸡马克丁）的销售页面，然后一边等 FL 官网给升级 DAW 搞打折活动[^3]。后来留意到 Producer Edition 马克丁最低应该是 1099，然后官网死活不给 DAW 本体打折。后续在公司赶版本加班的这段时间，部分项目需要音效也找我做[^4]。鉴于 fruity 版的音频编辑功能是“不存在的.jpg”，所以我就趁某次看到马克丁打折到我见到过的最低价的时候把高级版本给买了。

给公司做音效用的是工作时间，所以我没啥意见，DAW 我是自费买的，尽管我还另花三十让马克丁开了个专票以为公司可以把税给我抵了，然而结果是并不能...

## 今年值得一提的部分

我下半年的业余时间已经因为加班被占的几乎啥都不剩了，于使下面的重头戏当然是加班咯：

### 加班

尽管早就知道设计做了下个大版本的规划，甚至出了设计图，但最初的大版本规划则是把目前的项目（没新项目）搞到新规划要搞个一年多才能搞的差不多，但后来公司就突然决定要加班搞，赶版本，甚至招了很多人一起来搞，还多了一堆新项目，还初步定了个 19 年年底的 DDL。

时间定下来，事情也定下来，就风风火火的开搞了，当然在我来看这怎么都不会是什么好兆头。项目决定先不对社区开放，于使开发就从 GitHub 转到了公司内部的代码仓库，社区的 PR 和反馈就再也没人能响应了[^5]，至于先是要求按单双轮休（大小周）上班，并且二四六还要强制加班到晚上九点，然后开始传公司似乎要打算搬走了，然后公司仅提前了不到一天来通知我们要搬公司了。公司搬走后全员挤在一层上办公，通勤时间则从原本步行一刻钟出头一下提到一个小时，上班时间还能享用冬天份的冷风空调，还时不时得享受一下索尼隔音耳机都罩不住的装修噪音，时不时还要闻一闻楼上装修从通风口跑下来的~~甲醛~~油漆味儿。装修刚完事儿没多久，公司又让我们直接搬上去享受清新的油漆味。后来公司掀起了一股感冒发烧的狂潮（包括我），也有不少人买了 3M 的防毒面罩来戴，然后在我很好奇公司会不会放假个几天来除甲醛的时候，公司还要求项目组九九六赶进度赶一周——然后又续了两周的九九六——然后又回到了大小周的常规强制加班模式，直到今天还没有结束。

长期的加班使得我晚上回到住处后就基本不剩多少时间可以用来给自己“充电”或者休息娱乐了，下班之后甚至失去了打开 IDE 或者 DAW 的动力。早上的通勤时间迫使我不得不提前晚上的睡觉时间，尽管我仍然一贯的辣鸡睡眠质量导致我很难提前睡着——于使早上就日常睡不够了。后来我不得不选择利用上班期间的空挡时间来做“充电”和娱乐，由于中午用键盘会吵到趴着睡觉的同事，所以中午我就拿来看看视频[^6]，晚上吃饭的时间则拿来刷刷题或者写点解决自己需求的小东西[^7]。

依然记得是 19 年三月份左右，996.icu 的事情突然在网上疯传，同事也会庆幸和反对这种工作制。

现在，我晚上得早点睡。

### 大学以来最严重的一次生病

于是上面提到的生病狂潮中，我也发烧了。虽然最初我以为只是感冒才浑身无力也就没太在意[^8]，结果买了点药吃但两三天也没见好转，直到后来实在受不了才请假回住处歇着，睡到快晚上实在难受才去缺德地图上搜了个最近的（实在没力气跑远）诊所，一量体温38度+。后来吊瓶挂上搞到平常加班快下班的点才打完，然后回到住处给家里联系（惯例），还得忍着咳嗽和生病的难受不让家里发现。打吊瓶的医生建议我下一天接着打吊瓶，但由于第二天按规定得加班到九点，而医院九点关门，所以我也就没去接着吊瓶了。当然还有个蛋疼事，诊所给另开的药，其中有一盒让我一定要吃完别剩下的一盒，说是很贵两百多，第二天竹子帮我查了一下说京东上很便宜，以及说没必要开那个药[^9]，行吧人生第一次被诊所宰，那个诊所还不能刷医保...

由于扁桃体发炎导致的咳嗽后续愈演愈烈，最后咳嗽的事还是被家里发现了，然后家里让我注意卫生避免发烧（然而已经烧过了...），但咳嗽却很长一段时间都没见明显好转。我自己猜测是和公司装修的味道没散完多少有点关系，但我也不知道到底是不是有关系就是了。不过无论如何，无论按持续的时间长短，还是发烧的体温温度，还是咳嗽的程度，还是花出去的医药费，这应该是我从上大学到现在最严重的一次生病了。当然也没人给我个成就（

有一点比较后悔，假如能早点意识到我是高烧，我可能就能直接自己吃尼美舒利而避免挂吊瓶和被诊所宰了。

### 996 后感

加班期间一共有要求 2.5 周的 996 ，于是算是切身体验了 996 到底是什么感觉。单双轮休与隔日加班的混搭期间我就觉得不加班的那个晚上是个必要的缓冲期，还能每两天就缓口气（虽说即便是缓口气的时间也再没有动力在下班之后打开 IDE 或者是 DAW 了），而 996 则是整个周除了吃饭睡觉就都要投入到工作里去，容不得缓冲那样。

现在每天都要浪费两个小时在通勤上，想想如果是在大城市[^10]的话，通勤还要更长的时间，那么除了通勤和上班外，真的就完全不剩一丝时间给自己了。加上 996 之前的单双轮休+隔日加班期间，生病一次尚且都腾不出时间去看病，于是也更加能够理解最初 996.icu 风潮为什么会有如此大的反响——这样的生活，谁能“享受”？

### 在 Windows 下进行（Qt）开发

使用 Windows 的一大原因是打游戏，另一大原因是用 DAW ，因为我一直觉得在 Windows 下做（非 C# .net 之类的）开发非常不爽，所以只要想写什么东西我通常都会切回 Arch 里搞。而鉴于上面说的加班情况，我加班的这半年多几乎下班之后就基本不会切到 Arch 里去了。然而有的需求还是很蛋疼，比如上面也提到过的，我没法直接用 Windows 来查看我手机上未读的通知，于是我就有需求把 kdeconnect 给搞到 Windows 上来用。于是后来发现 KDE Craft 是个还不错的东西，除了 IDE 我没折腾好用 Craft 环境之外，这东西还不错。能用 pkg-config ，有能用 ECM 的 cmake ，甚至有 auto\*\*\*\*，于是给移植一些应用之类带来了更方便的体验。不过目前的情况，还是得看看怎么配好 IDE 才能更舒服一些，以及目前看上去 Windows 上的 DBus 好像很容易抽风的样子。

不过老实说，常规的 Windows 下进行这种开发似乎会用 canon 之类的东西做依赖管理，不过这些东西我完全没玩过，所以其实对于 Windows 下进行这种开发的常规操作还是摸不清头脑。

## 最后

于是想来想去似乎也没有别的什么可写的了，下半年基本就是加班加班加班，上半年做了什么现在甚至有点想不起来了，总之似乎都没什么特别值得一提的事情，于是感觉我的 2019 一整年都挺失败的。

于是希望 2020 年能改变这种现状，过的更理想一些吧。

现在是 2020年1月5日22点51分，还有点别的事情加上需要早睡，所以今天大概写不了 2020 时间囊了，争取明天把它写了。

就是这样。
